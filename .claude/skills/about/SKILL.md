## What is APEX?

**APEX** (Agent-Powered EXcellence) is a Claude Code framework that turns "build me X" into a shipped, tested, documented application вҖ” autonomously. You tell it WHAT to build, it handles HOW.

## How It Works вҖ” The Autonomous Pipeline

You say "build me X". APEX runs the entire pipeline. You approve at 3 gates:

```
"Build me X"
  в”Ӯ
  в”ңв”Җ PHASE 1: PLAN в”Җв”Җв”Җв”Җв”Җв”Җ /prd auto-generates requirements
  в”Ӯ   вҸё GATE: Approve the PRD
  в”Ӯ
  в”ңв”Җ PHASE 2: ARCHITECT в”Җ /architecture designs the system
  в”Ӯ   вҸё GATE: Approve the blueprint
  в”Ӯ
  в”ңв”Җ PHASE 3: BUILD в”Җв”Җв”Җв”Җв”Җ Agent teams work in parallel
  в”Ӯ   (Watcher monitors, Builders code, QA verifies)

  в”ңв”Җ PHASE 4: QUALITY в”Җв”Җв”Җ /qa + /security + /a11y + /cx-review

  в”ңв”Җ PHASE 5: SHIP в”Җв”Җв”Җв”Җв”Җв”Җ Code review plugin + Technical Writer
  в”Ӯ   Auto-version: [Unreleased] вҶ’ semver after merge
```

### Quality Gates

Nothing ships without passing:
1. No `any` in TypeScript
2. No `console.log` in production
3. Functions вүӨ 30 lines, files вүӨ 300 lines
4. Conventional commits (72-char subject)
5. Design tokens only вҖ” no hardcoded colors
6. Mobile-first + dual theme from day one
7. Lazy routes, virtualized lists, no N+1 queries

## How to Use APEX Effectively

### Starting a new project
Just say: **"Build me a [type of app]"**. APEX handles the PRD, architecture, and implementation. You approve at gates.

### Quick fixes
Say: **"Fix this bug"** or **"Change X to Y"**. No pipeline needed вҖ” APEX just does it.

### Shipping
Say: **"Ship it"**. APEX branches, commits, pushes, creates PR, runs code review, and waits for your merge approval. After merge, auto-versions the release.

### Learning
Say: **"Teach me about [concept]"**. APEX explains with context tailored to your level вҖ” from terminal basics to system design.

### Key principles
- **Never ask which command to run** вҖ” just say what you want
- **The pipeline is invisible** вҖ” you only see the 3 approval gates
- **Quality is non-negotiable** вҖ” nothing ships untested
- **Design matters** вҖ” read Design DNA before building UI
